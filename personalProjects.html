GregBuild - Build System for C/C++ Projects (January 2021 to April 2021)

Project Description

A build system for C/C++ projects that allows the user to build for either the host machine and or a different target hardware. For example, using their x86 laptop, a user could build an executable for that same x86 laptop. Or for the case of building for a different target hardware, the user could use GregBuild on their x86 laptop to build an exectuable to run on a different harware, such as a microcontroller.

GregBuild is a project that I see myself working on for a long time. There is a lot that goes into a professional grade build system, and this project is not there yet. I worked on this for about four months before making the 0.0.1 release. After those four months it was working well enough that I decided to stop actively developing it and start using it as a user.

There are a lot of things I would still like to improve. So another 4 months of working on GregBuild may be in the cards for the not so distant future.

Helping Hands Routine Manager (March 2020 to October 2020)

Project Description

This project came out of the "Helping Hands Café Systems" project below. Myself and a partner were working together to help special education teacher's organize a student run café at a high school in Illinois where his family lived. He managed the organization/communication with the school and I managed the technical side of things.

When COVID-19 began and schools were shut down for a while, we stopped working on the café idea and began working on a new idea to help the teachers while students were at home. The idea was to have a web application that managed a set of "routines" for each student to complete at home. After the student completed each step they would click the "Next Step" button and progress through the set of steps that made up a routine.

The teacher's could make unique routines and assign those routines to students. The application also had a page called the "Live Feed" that was a real time tracker of the students' progress through their routines. We thought this could be useful for teacher's to see where the students where in a routine if the teacher was working with the student over the phone or through zoom/skype at the same time.

The last feature that this routines manager app had was that it would record how long it took the student to complete each step and each routine. Collecting this data would help monitoring learning and progress of the student completing their routines. We thought this would be useful data for the teacher's and occupational therapists to have while working with the special education students.

This project reached a few teachers who made accounts for their classrooms.

Helping Hands Café Systems (September 2019 to March 2020)

Project Description

Helping Hands Café Systems was a project with the goal to improve the efficiency of student run café programs in schools. More specifically, it was aimed at café programs where the students are enrolled in the school's special education program and the café organizers are occuptional therapists.

The inspiration for Helping Hands came from a University of Calgary track teammate, Jan Erik Naess. Jan Erik discovered this problem through the school where his mom was working and came to me in the fall of 2019 asking me to help him develop a solution. The only technology the program had in place at the time was a Google Form that allowed customers to place orders. Once orders were placed occuptional therapists would print of large summaries and then filter go through all of the orders and data manually. This was very hard and time consuming. To help this I developed a web application and an automatic label printing solution.

In January 2020, we got our system set up at Hinsdale South High School in Darien, Illinois. We were hoping to expand to some of the nearby schools, but ended up stopping on the project when COVID-19 started.

In addition to the new technology, we have made a few posters for the students and teachers of the café to put up around the school.

The solution we implemented was created was a simple google scripts app. We used this technology to start because it was able to easily interface with the pre-existing Google form that the café was using. Before we finished working on this project, I started to create a more invloved application using the Java Spring framework and VueJs. That application had customer accounts, an improved order form, and a performance measurement system to track how the students were progressing with their skills in the café. We hoped that recording data on how long it took students to make orders would be beneficial to the occupation therapists working with them and could even perhaps be used on a resume for the students to apply to jobs at cafés outside of the school. However, when COVID-19 started we switched to working on Helping Hands Routines and the Java Spring/VueJS café application never made it to the school.

Lessons Learned

I learned a bit about Google's developer products, Windows batch scripting, and Brother label printers.

Google has a cloud print service. However, we had formatting issues when using this to send print jobs to the label printer. So the solution to automate printing was to use batch scripting and some python to download data from a Google drive, open printing software, and print orders with a preformatted template. This was done with a Windows laptop as the 'print server'. During this process, I learned that Brother's QL series label printer support the use of raster and ESC-P commands. Although never implemented, an idea for later improvements was to send raw binary data directly from our application to the printer without needing to open the Brother QL printer software on a desktop computer. We were also considering the switch to using a small raspberry pi device as a print server instead of a Windows laptop.
Dinos Inspire Training Log Website (June to August 2017)

Project Description

This was a weekend warrior type project that I worked on during the summer of 2017. This project was a web-based application, built using PHP, that allowed distance runners to create accounts and then store data related to their training in a database. The inspiration for this project came from my involvment in the sport and awareness of the importance of keeping a proper training log. I viewed the project as an opportunity to learn about the technology and get a sense for how other training log websites, like Garmin Connect or Strava, might work.

This project did not reach completion by the end of summer 2017 and got put on the backburner when school begin in September. However, I did feel that I took this project far enough to achieve the goal of learning the basics of how client-side and server side languages work together in a web application to navigate from page to page, pass data, and interact with a database.

Python Texting App (August 2017)

Project Description

This was another project for fun in the summer of 2017. The idea for the project came from a youtube video. The video was about 20 minutes long and showed how to use the Twilio API to send an SMS message with python.

To expand on what was shown in the video, I decided to add a password system that identified the user an sent the message to a different phone number based on the user. I also decided to add a user interface. To do this I used the Tkinter python library, and this was my first introduction to graphical user interfaces. This little app also added to the ideas presented in the video by using ngrok webhooks to allow for the phone that recieved the initial message to send a response back to the computer.

Lessons Learned

The main take aways from this project were learning about APIs, webhooks, and graphical user interfaces. This project introduced me to the idea of sending requests from one machine to another, and seeing how this can be triggered by event based programming.
