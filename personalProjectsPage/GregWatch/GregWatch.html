GregWatch - Running Watch (April 2021 to August 2021)

Project Description

A running watch.

The github repo for this project is here: https://github.com/GregoryOrd/GregWatch


One morning I woke up and started thinking about logging my training and how I had lost touch with that habbit. Previously, I had logged training data with pen and paper. This took a few minutes each day and, over time, became harder and harder to look through. With years of data spread over different physical log books, I wished that I had a newer watch that could automatically upload run data to your phone and into applications like Strava or Garmin Connect where you could analyze it easily. My current running watch is an old Garmin and I haven't been able to get run data off of it in recent years.

I was also looking for a new side project at the time and decided it would be more fun to try to build my own GPS running watch than go buy a watch similar to the watches all of my running friends have. I thought this project would be a good opportunity to learn more about some of the real world problems faced by embedded systems and wearable electronics.

This project idea was the main motivation behind the GregBuild project. I needed a build system for writing embedded C code to go on the microcontroller in the watch. As has become the theme of my recent side projects, I decided to write my own build system (for learning and to tailor it to my own specific needs and wants). So I created GregBuild. After about four months of getting GregBuild to a level that was sufficient and convenient enough that I thought I could use it for a real project I began working on the GregWatch project.

Progess to this date has included getting a simple timer to run on the screen with a button to stop and start it. This work has also revealed a few bugs in GregBuild and is a good opportunity to gain knowledge about that project as I work on this one. Eventually, I could tie this project into the DinosInspire training log project from a few years back. A lot of work needs to be done before I start to work on that, but it would be an interesting integration of a bunch of different projects to create a full product similar to how companies like Garmin build everything from the watch, up to the web user interface and all of the data analysis of what is collected from the physical device.

Lessons Learned

In addition to the code, I needed to learn a lot of other things about creating electronics. Some of the lessons learned and lessons still to learn include:

  • How to solder.
  • How to design a custom PCB.
  • How to use KiCAD Software for PCB design.
  • How to transmit data using the SPI interface (reviewed/relearned from school - ENCM 511 - Embedded Systems Interfacing)
  • How to generate basic pixel graphics on a simple screen
