GregBuild - Build System for C/C++ Projects (January 2021 to April 2021)

Project Description

A build system for C/C++ projects that allows the user to build for either the host machine and or a different target hardware. For example, using their x86 laptop, a user could build an executable for that same x86 laptop. Or for the case of building for a different target hardware, the user could use GregBuild on their x86 laptop to build an exectuable to run on a different harware, such as a microcontroller.

The github repo for this project is here: https://github.com/GregoryOrd/GregBuild


GregBuild is a project that I see myself working on for a long time. There is a lot that goes into a professional grade build system, and this project is not there yet. I worked on this for about four months before making the 0.0.1 release. After those four months it was working well enough that I decided to stop actively developing it and start using it as a user.

There are a lot of things I would still like to improve. So another 4 months of working on GregBuild may be in the cards for the not so distant future.

  • Link order can matter.
  • Debugging on microcontrollers is more difficult than in applications running on your laptop.
  • First big project using C. Learned/refreshed my memory on a lot of little things about the language and memory management.
  • Discovered Windows Subsystem for Linux
  • Learned how to use valgrind (Saw/heard about it on internship, but didn't get much exposure to it until using it with this project).
  • Got a better understanding for the compilation/linking process. Learned about some cool tools on linux like 'nm' for inspecting binary files.
  • Learned a little bit about name mangling in C++. Discovered the c++filt tool for demangling names.
  • Got a feel for some of the problems that need to be solved by a test framework.
  • Learned about dynamic linking and libary file types (.dll/.so).
  • Wrote my first application using a plugin architecture.
  • Learned a little bit about releases and the ideas behind them while on internship. Created my first at home release of a project using Github.
  • Applied a lot of the lessons learned while on internship to this project. (Clang-format, git submodules, build system/test framework knowledge, coding style, readability, code modularity, screaming architecture/repo structure)
  • Setup clang-format with VS Code.
  • Learned about VS Code. Learned how to add files to the VS Code include path so the editor doesn't have issues with unknown header files.
