Linux Phone and Home Network Ecosystem

Project Description

Recently I have become aware that there are some linux phones out on the market. From the research I have done, it seems like these phones may not work as well as an Android or iOS phone, and there is a much smaller selection of apps. However, I am still intrigued and think I might try one in the future. Some of the phones I have seen are security focused and have features like hardware kill switches for the camera, microphone and cellular modem. These security features along with the ability to customize the linux system and run my own apps on the phone makes it a mobile phone like this a very different offering than a traditional Android or iOS phone.

If a linux phone setup was acheived it might be possible to run clients from my previous projects like GsMessaging, GregCall, and a ReceiptReader app on the linux phone. If I pair this with improvements my home networking setup, to do things like expose endpoints or setup a VPN (as mentioned in the Raspberry Pi Hole/Access Point project or GSMessaging projects), I could route traffic run the phone into my home network even when I am away from home.

With a mobile device that I could control and a nice home network setup, I could have my own little ecosystem of apps and infrastructure!
