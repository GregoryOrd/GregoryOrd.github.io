LinuxKernelBlink - A Linux Kernel Module for Controlling GPIO Pins and Blinking an LED (December 2021)

Project Description

This wasn't a full project, but rather a small exercise (following a youtube tutorial) to learn how to write a Linux kernel module.

My github repo for this exercise is here: https://github.com/GregoryOrd/HosuingMarketMonitor though much of this code in this repo was copied from the tutorial author.

This kernel module was written to expose a GPIO driver on the procfs filesystem. The interface allowed a userspace program to write a line to the file with format "%d,%d". The first integer respresented the GPIO pin number and the second integer represented the value to set that pin to (0 or 1). With an LED connected to the GPIO pin, this allowed the userspace program to control the LED by writing to the procfs file. As writes to this procfs file were happening, the kernel module would use printk to log the events to the kernel log, which could be viewed with the dmesg command.
