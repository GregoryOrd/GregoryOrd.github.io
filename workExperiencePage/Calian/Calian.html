Calian Advanced Technologies (May 2022 to Present) (Remote)

Calian Advance Technologies

From the Calian Website: "Calian Advanced Technologies is a global supplier of highly technical solutions, services and products to the aerospace and defence, satellite, wired and terrestrial wireless, test and measurement, agricultural technology, GNSS technology and nuclear industries."

This was my first experience working in a linux only environment. I learned a decent amount here about linux tools, software packaging, firewalls, selinux, network interfaces, and systemd. I was even the main contributor on the effort to upgrade the servers for a project from an old RHEL version to a new one after it went end of life.

The Gateways Team

The team I was on at Calian was a small team called the "Gateways" team. We had various projects, but they were all related to the operation of Satellite Ground Gateways. In particular, our gateways handled aeronautical data and the main purpose of our systems was to connect the ground network to the satellite network. ACARS (Aircraft Communications Addressing and Reporting System) and ADS (Automatic Dependent Surveillance) data are some example of the types of data that went through these gateways. Having redundant systems was a common theme for these projects and we had servers at multiple sites around the world.

Support Tasks

In addition to the above tasks, my role at Calian sometimes involved support tasks and bug fixes for issues that came up at the various production sites. These tasks typically involved reading through log data, analyzing messaging sequences, and trying to debug the systems from a distance. In some instances we even created tools to capture data for debugging at a later date. One example was creating a python program that used inotify to monitor a directory that was receiving files from an SFTP transfer and stream a copy of the files into a tarball in another directory before the server's application code could process and delete them. This program allowed us to capture the files and analyze them as part of the debugging process for the particular issue that was happening in production.

How I Grew My Skills

Throughout this experience I grew my programming skills in C++, Java, Python, and Bash. But more importantly I learned a lot about all of the things that need to be present in an environment or on a system rather than just the application code. I improved my understanding of databases and networking, and gained experience handling situations where all you have is some logs or data from the production system but you are unable to reproduce the issue in a test environemnt. There is still lots to learn, but I now have a much better grasp of linux system and production server concepts.
