For one project, I was tasked with adding support for using a non-default truststore and keystore for outgoing TLS interfaces. These interfaces included connecting to an LDAP server and various database servers. The client libraries we were using already had support for TLS, and the code change to use a non-default truststore and keystore was quite trivial. Where this became interesting was in testing.

To test, I needed to configure all of the test databases to require TLS connections, and I needed to generate and configure TLS certificates. There was nothing revolutionary in this task, but it was a good opportunity to learn a little bit about TLS certificates and how java handles TLS.
